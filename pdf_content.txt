1. Executive Summary
Stego-Hunter is a full-stack, microservices-based cybersecurity platform for detecting and neutralizing steganographic payloads hidden in digitalimages and deep neural network (DNN) model files.
The platform combines:
Multi-method detection: A CNN classifier (trained on the Kaggle Stego-Images-Dataset) combined with statistical steganalysis engines(Chi-Square, RS Analysis, SPA) and YARA signature matching, orchestrated as a weighted ensemble.Active neutralization: An engine that removes detected payloads from infected files while preserving visual/functional integrity, verifiedthrough SSIM, PSNR, and re-scan loops.DNN model defense: Detection and sanitization of steganographic payloads embedded in neural network weight parameters.Production-grade infrastructure: JWT authentication, async job processing, real-time WebSocket updates, and full observability.
Key Data Sources

2. Problem Statement

2.1 The Threat
Steganography is the practice of hiding data within seemingly innocent carrier files. Unlike encryption (which makes data unreadable),steganography makes data invisible. This presents critical security challenges:
Malware delivery: Attackers embed malicious JavaScript, PowerShell scripts, or binary executables inside images that bypass traditionalsecurity scanners.Data exfiltration: Sensitive corporate data can be smuggled out hidden inside innocuous image uploads.Covert communication: Command-and-control (C2) instructions can be embedded in images shared on public platforms.DNN supply chain attacks: Model files downloaded from public repositories (HuggingFace, Model Zoo) can carry hidden payloads in theirweight parameters without affecting model accuracy.
2.2 Current Gaps
Gap ImpactMost security tools don't scan for steganography at allHidden payloads pass through firewalls and DLP systems undetectedExisting stego-detection tools use single methodsHigh false-positive rates; miss advanced embedding techniquesNo tools offer neutralization Detection alone is insufficient; infected files remain dangerousDNN model files are never scannedFastest-growing attack vector is completely ignoredNo enterprise-grade stego platforms existExisting tools are research scripts, not production systems
2.3 Solution
Stego-Hunter addresses every gap with a unified platform that detects, reports, and neutralizes steganographic threats across both images and DNNmodels.

5.3 DNN Defense Service
Attribute DetailPort 8002TechnologyFastAPI + TensorFlow + PyTorch + ONNX RuntimeResponsibilitiesParse DNN model files, analyze weight distributions for anomalies, detect steganographic embedding in model parameters
Why This Service Exists:
Attackers can hide up to ~17 MB of data in a 100 MB model file by modifying the least-significant bits of float32 weight parameters — withoutmeasurably affecting model accuracy. This service detects and neutralizes such attacks.
Supported Formats: .h5, .pt, .pth, .onnx, .safetensors, SavedModel
5.4 Neutralization Service
Attribute DetailPort 8003TechnologyFastAPI + Pillow + OpenCV + NumPyResponsibilitiesRemove detected steganographic payloads, verify post-neutralization integrity, generate before/after comparisons
This is the core differentiator of Stego-Hunter. See Section 8 for full architecture.

11. Database Schema
11.1 PostgreSQL — Core Tables
-- ═══════════════════════════════════════ -- USERS & AUTH -- ═══════════════════════════════════════ CREATE TABLE users (     id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),     email           VARCHAR(255) UNIQUE NOT NULL,     username        VARCHAR(100) UNIQUE NOT NULL,     hashed_password TEXT NOT NULL,     role            VARCHAR(20) DEFAULT 'analyst',     is_active       BOOLEAN DEFAULT TRUE, 
    created_at      TIMESTAMPTZ DEFAULT NOW(),     updated_at      TIMESTAMPTZ DEFAULT NOW() ); 
CREATE TABLE api_keys (     id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),     user_id         UUID REFERENCES users(id) ON DELETE CASCADE,     key_hash        TEXT NOT NULL,     name            VARCHAR(100),     scopes          TEXT[],     expires_at      TIMESTAMPTZ,     created_at      TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- SCAN JOBS & RESULTS -- ═══════════════════════════════════════ CREATE TABLE scan_jobs (     id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),     user_id          UUID REFERENCES users(id),     status           VARCHAR(30) DEFAULT 'pending',     job_type         VARCHAR(20) NOT NULL,     file_name        VARCHAR(500),     file_hash_sha256 VARCHAR(64),     file_size_bytes  BIGINT,     mime_type        VARCHAR(100),     minio_path       TEXT,     priority         INTEGER DEFAULT 5,     created_at       TIMESTAMPTZ DEFAULT NOW(),     started_at       TIMESTAMPTZ,     completed_at     TIMESTAMPTZ,     error_message    TEXT ); 
CREATE TABLE scan_results (     id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),     scan_job_id       UUID REFERENCES scan_jobs(id) ON DELETE CASCADE,     is_stego          BOOLEAN NOT NULL,     confidence        FLOAT NOT NULL,     detection_methods JSONB,     threat_level      VARCHAR(20),     payload_map       JSONB,     feature_vector    JSONB,     yara_matches      JSONB,     metadata          JSONB,     created_at        TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- NEUTRALIZATION -- ═══════════════════════════════════════ CREATE TABLE neutralization_jobs (     id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),     scan_result_id  UUID REFERENCES scan_results(id),     user_id         UUID REFERENCES users(id),     status          VARCHAR(30) DEFAULT 'pending',     strategy_used   VARCHAR(50)[],     original_path   TEXT NOT NULL,     sanitized_path  TEXT,     created_at      TIMESTAMPTZ DEFAULT NOW(),     completed_at    TIMESTAMPTZ ); 
CREATE TABLE integrity_reports (     id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),     neutralization_id       UUID REFERENCES neutralization_jobs(id) ON DELETE CASCADE,     ssim_score              FLOAT,     psnr_db                 FLOAT,     mse                     FLOAT,     perceptual_hash_match   BOOLEAN,     pixel_diff_percent      FLOAT,     model_accuracy_before   FLOAT,     model_accuracy_after    FLOAT,     verification_scan_clean BOOLEAN,     quality_approved        BOOLEAN,     details                 JSONB,     created_at              TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- REPORTS & AUDIT -- ═══════════════════════════════════════ CREATE TABLE reports (     id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),     scan_job_id       UUID REFERENCES scan_jobs(id),     neutralization_id UUID REFERENCES neutralization_jobs(id),     user_id           UUID REFERENCES users(id),     report_type       VARCHAR(30),     format            VARCHAR(10),     minio_path        TEXT,     created_at        TIMESTAMPTZ DEFAULT NOW() ); 
CREATE TABLE audit_log (     id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),     user_id       UUID REFERENCES users(id),     action        VARCHAR(100) NOT NULL, 
    resource_type VARCHAR(50),     resource_id   UUID,     ip_address    INET,     user_agent    TEXT,     details       JSONB,     created_at    TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- THREAT INTELLIGENCE -- ═══════════════════════════════════════ CREATE TABLE threat_signatures (     id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),     name        VARCHAR(200) NOT NULL,     category    VARCHAR(50),     yara_rule   TEXT,     description TEXT,     severity    VARCHAR(20),     source      VARCHAR(100),     is_active   BOOLEAN DEFAULT TRUE,     created_at  TIMESTAMPTZ DEFAULT NOW(),     updated_at  TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- WEBHOOKS -- ═══════════════════════════════════════ CREATE TABLE webhooks (     id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),     user_id   UUID REFERENCES users(id),     url       TEXT NOT NULL,     events    TEXT[],     secret    TEXT,     is_active BOOLEAN DEFAULT TRUE,     created_at TIMESTAMPTZ DEFAULT NOW() ); 
-- ═══════════════════════════════════════ -- INDEXES-- ═══════════════════════════════════════ CREATE INDEX idx_scan_jobs_user     ON scan_jobs(user_id); CREATE INDEX idx_scan_jobs_status   ON scan_jobs(status); CREATE INDEX idx_scan_jobs_hash     ON scan_jobs(file_hash_sha256); CREATE INDEX idx_results_threat     ON scan_results(threat_level); CREATE INDEX idx_neutral_status     ON neutralization_jobs(status); CREATE INDEX idx_audit_user         ON audit_log(user_id, created_at DESC); CREATE INDEX idx_audit_action       ON audit_log(action, created_at DESC);
